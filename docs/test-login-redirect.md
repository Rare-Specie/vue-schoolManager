# 登录后默认打开个人中心 - 测试说明

## 修改内容

### 1. 路由配置修改 (`src/router/index.ts`)
- 在 `/main` 路由添加 `redirect: '/main/profile'`
- 这样访问 `/main` 时会自动重定向到个人中心页面

### 2. 路由守卫修改 (`src/router/index.ts`)
- 所有重定向逻辑都改为跳转到 `/main/profile` 而不是 `/main`
- 包括：
  - 已登录用户访问根路径 `/` 时
  - 会话恢复成功时
  - token验证成功时

### 3. 登录页面修改 (`src/views/LoginView.vue`)
- 登录成功后跳转到 `/main`（会自动重定向到个人中心）

## 测试步骤

1. **正常登录流程测试**
   - 打开应用，进入登录页面
   - 输入用户名、密码，选择角色
   - 点击登录按钮
   - 预期结果：自动跳转到个人中心页面（/main/profile）

2. **已登录状态访问根路径测试**
   - 登录后，在浏览器地址栏输入根路径 `/`
   - 预期结果：自动跳转到个人中心页面（/main/profile）

3. **会话恢复测试**
   - 登录后刷新页面
   - 预期结果：保持在个人中心页面

4. **不同角色测试**
   - 分别用管理员、教师、学生账号登录
   - 预期结果：所有角色登录后都默认显示个人中心

## 预期效果

- 用户登录成功后，自动显示个人中心页面
- 个人中心显示用户的基本信息和快捷操作
- 菜单中"个人中心"项被高亮选中
- 用户可以方便地查看个人信息和进行常用操作
